# 후킹 라이브러리 (Hooks Library) — 큰틀 공감 + 권위 톤

> **1순위 = 큰틀 공감 후킹**(광범위 도달). 일주특정·요소특정은 본문 심화용으로만(첫 줄 금지).
> 후킹 공식(검증): **호기심 갭 + 광범위 Pain + 부드러운 행동명령**.
> 톤: **권위·확신**. 자기비하("내 사주도 못 읽었다")·"무료로 봐준다"·결과/시기 보장은 전면 금지.
> 첫 줄 250자 이하. 근거: socialchamp hook 공식 / wellnesswriter "문제를 깊이 이해한 시연이 권위를 만든다".

## 왜 큰틀 후킹인가 (도달 모수)
- 스레드·릴스·Shorts는 **다수 반응 = 노출**. 일주("경오일주…")로 첫 줄을 좁히면 공감 모수가 줄어 도달이 깎인다.
- 그래서 **첫 줄은 누구나 자기 얘기로 느끼는 큰 틀**(돈·관계·결정·흐름) → 본문에서 사주 구조로 좁혀 들어간다(유인 → 몰입 → 전환).

## 후킹 공식 (모든 첫 줄에 적용)
```
[호기심 갭] 당연해 보이는 것을 뒤집는 한 줄
 + [광범위 Pain] 특정 일주가 아니라 '많은 사람이 겪는' 상황
 + [부드러운 행동명령] 댓글/생년 남기기 (보장·압박 아님)
```

## 큰틀 후킹 5종 (1순위 · 매일 변형 사용)

### A. 돈·재물 (광범위)
- "분명 남들만큼은 버는데, 왜 손에 남는 게 없을까."
- 본문 심화(좁히기): 버는 자리 ≠ 지키는 자리 구조 → (일주·십성은 여기서 등장)

### B. 결정·타이밍
- "지금 움직여야 할지, 더 기다려야 할지 — 매번 이게 제일 어렵죠."
- 본문 심화: 사주엔 '큰 구조'와 '시기'가 따로 있다 → 자미두수 교차로 '때'를 본다

### C. 노력 대비 결과
- "열심히 사는데 이상하게 안 풀리는 시기, 분명히 있어요."
- 본문 심화: 흐름(대운·세운)이 어긋난 구간 → 무엇을 줄이고 무엇을 밀지

### D. 관계 반복
- "사람은 바뀌는데 끝은 늘 비슷하게 흘러간다면."
- 본문 심화: 관계의 '패턴 자리' → 명리 구조 + 시기 교차

### E. 흐름 전환 신호
- "운이 바뀌기 직전엔, 꼭 먼저 나타나는 신호가 있어요."
- 본문 심화: 흐름이 도는 구간의 징후 → 단정 아닌 '참고 신호'로

## 본문 심화용(첫 줄 금지) — 일주·요소 라이브러리
> 큰틀 후킹으로 유인한 뒤 **본문에서만** 사용해 전문성을 시연.
- 일주 대입: "예를 들어 경오일주라면, 버는 그릇은 큰데 지키는 자리가…"
- 숫자·구조: "돈이 새는 명식 패턴은 크게 3가지인데 2번이 제일 흔해요."
- 오해 정정: "백호살 있으면 사납다? 구조를 보면 오히려…"

## 오픈루프 마무리 (권위 톤 · 매 게시물 1개)
- "당신은 어느 쪽인가요? 댓글에 생년(또는 일주)을 적어주시면 흐름을 짚어 답을 달아드립니다."
- "본인 얘기 같다면 ㅇ, 아니면 ㄴ 댓글 주세요. 패턴별로 짚어드릴게요."
- "다음 편에서 '때'를 어떻게 읽는지 이어 풀어드립니다." (연재 오픈루프)
> 금지: "무료로 봐드릴게요"(격 하락) / 시간 약속("30분 내")은 팔로워 약속으로 넣지 않음.

## 게시 후 30분 자기답글 (알고리즘 수명 · 팔로워 약속과 분리)
- "참고로 같은 구조라도 월지·시지·세부 흐름에 따라 또 갈립니다…" 식 보충 1개.

## 금지 (banned-terms 일치)
- ✗ 보장·시기 단정: "반드시 / 100% 적중 / 한 달 안에 / 확실히 / 완치 / 수익보장 / 무조건"
- ✗ 자기비하 톤: "제 사주도 못 읽었다 / 답답해서" (권위 파괴)
- ✗ "무료로 봐준다" 류 (격 하락) — 가치는 '깊이의 시연'으로
- ✗ 일주·요소로 시작하는 첫 줄 (도달 모수 축소) — 본문 심화로만
- ✗ 자격증 명시: "심리상담사 1급 / 명리상담사"

## 운용 (측정→교정)
- 매일 큰틀 후킹 A~E 중 **≥5개 주제**를 변형 게시 → 주제별 도달/조회/답글 측정.
- 그중 **파급력 1위 주제 1개**를 릴스·Shorts·캐러셀로 확산(`automation/repurpose.py`).
- 주제별 성과는 `tracking/daily-tracker.xlsx`의 `topic`·`주제별성과`로 롤업 → 승자 더블다운.
- 후킹 생성: `automation/daily_hooks.py`(매일 ≥5 주제) → `automation/repurpose.py`(선택 주제 1→11).
