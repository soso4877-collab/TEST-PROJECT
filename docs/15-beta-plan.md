# 15. 지인 베타 계획 (Phase 0~9 완료 후 검증 단계)

> 목적: 기능은 완비됐으나 실주문 0건. 추가 빌드 전에 **실제 사람 1~2명**에게 풀이를 돌려
> 반응·풀이 품질을 수집하고, 다음 빌드 우선순위를 가정이 아니라 사실로 정한다.
> 핵심 리스크(프리모템): 피드백이 막연하면 백로그만 늘고 결정 못 함 → **구조화 양식으로 방어**(§4).

## 1. 대상·범위
- 대상: 신뢰 가능한 지인 1~2명(솔직한 피드백 가능). 실명 동의·생년월일 제공 동의 필수.
- 범위: 통합 상품 1건/인. 시진불명·음력 입력 등 변형은 대상이 해당될 때만.
- 개인정보: 베타 종료 후 주문은 파기(제21조) — `delete_order.py`. 엔진 가치는 익명 추출
  (`insight.py --extract-insight`)로만 보존. 베타 안내 시 "검수 후 파기" 고지.

## 2. Intake (운영자)
대상에게서 받을 것: 이름(호명용)·생년월일·**출생시각(모르면 '미상')**·양/음력·성별·고민 1개(선택).
출생지는 서울 기본(다르면 경도/위도).

## 3. 실행 → 검수 → 발송 (명령)
검수 화면 경로(권장):
```
./.venv/Scripts/python.exe -m uvicorn sajugen.app:app --host 127.0.0.1 --port 8765
# http://127.0.0.1:8765/admin 에서: 접수 → (자동)계산·초안 → 검수 → 승인 → 발급
```
또는 CLI 단건:
```
./.venv/Scripts/python.exe -m sajugen.cli gen --birth "1990-05-20 14:30" \
  --gender 남 --name 홍길동 --horoscope 2026-06-01 --concern "올해 이직" --out beta1.pdf
# 음력: --lunar [--leap] / 시진불명: 시각 생략("1990-05-20")
```
- 승인(APPROVED) 전 발송 금지(상태머신 강제). 발급 PDF를 운영자가 **직접 전달**.
- 발급 전 검수에서 호명·기준연도·금지표현·말투를 최종 확인.

## 4. 피드백 양식 (대상에게 — 막연함 방지가 핵심)
풀이를 읽은 직후 받는다. 7문항, 5분.
1. **가장 정확/와닿은 부분** 1곳과 이유(섹션명 또는 문장 인용).
2. **틀렸거나 막연했던 부분** 1곳과 이유.
3. 분량: 너무 길다 / 적당 / 짧다.
4. 말투·톤: 신뢰감 든다 / 보통 / 어색하다 — 어색하면 어디가.
5. 디자인·가독성(표지·본문·한지): 좋다 / 보통 / 별로.
6. **돈 낼 의향이 든다면 적정가**(원). 안 든다면 그 이유.
7. 지인에게 **추천 의향**(0~10)과 한 줄 이유.
> 자유서술 1줄: "이 풀이가 ___였다면 더 좋았다."

## 5. 성공 기준 / 산출
- 정량: 추천 의향 평균·적정가 분포·"정확" vs "막연" 지목 섹션 집계.
- 정성: §4의 1·2·7번을 백로그 항목으로 환산(섹션별 개선 우선순위).
- 산출: 베타 종료 시 STATE.md에 "베타 N건 요약 + 다음 우선순위 Top 3" 기록.
  주문 파기 + 익명 추출(`delete_order --extract-insight`) 완료.

## 6. 지금 시작하기
1. 운영자가 지인 1~2명 섭외 + 생년월일 등 수집(§2).
2. 그 정보를 주면 운영자와 함께 /admin 또는 CLI로 1건 생성 → 검수 동선 점검.
3. 발급·전달 후 §4 양식으로 피드백 수거 → §5로 정리.
