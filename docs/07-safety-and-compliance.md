## 개인정보
- 수집 최소화: 이름(닉네임 가능)·생년월일시·성별·출생지·고민 텍스트. 연락처는 발송 채널에만 두고 DB 비저장(MVP).

- 로그 마스킹: 생년월일 YYYY-**-** 형식, 고민 텍스트 로그 비기록.
- LLM 입력 최소화: 생년월일 원본·출생지 비전달, 파생 계산값만 (docs/04 원칙).
- 삭제 요청: scripts/delete_order.py — order_id 기준 PDF·DB·로그 일괄 삭제 (Phase 8).

## 환불·오류 대응 (초안)
1. 계산 오류(원국·명반 틀림) 확인 시: 정정 재발행 + 전액 환불 우선.
2. 문안 불만족: 1회 무상 재생성(검수 포함). 재생성 이력 audit_log.
3. 검수 누락 발견: 즉시 회수 공지 + 재발행.
4. 표시광고법·플랫폼 정책 변경 시 본 문서·고지 문구 우선 갱신.
