## 2. 만세력 3원 교차검증

- 구성: lunar-python(1.4.8 고정) ↔ Skyfield(de440s) ↔ KASI 캐시.
- 대조 항목: (a) 음↔양 변환+윤달, (b) 일진 간지, (c) 절입 시각(Skyfield↔KASI kst, 허용오차 2분), (d) 연간지(세차).
- KASI 월건은 음력월 간지 → 사주 월주(절기 기준)와 다를 수 있음. 월주는 절입시각+생시 자체 판정, 월건은 참고 대조만.
- 캐싱: 실시간 API 의존 금지. scripts/kasi_dump.py로 1900~2050 전수 → data/kasi_cache.sqlite. 절기 연 1회 증분.

### 2-1. Phase 1 실측 결과 (2026-06-10, 키 발급·전수 캐싱·전 구간 교차 완료)
- 캐시 적재: 음양력 55,152일(1900-01-01~2050-12-31), 절기 672행(2000~2027 전 연도 24절기). 파일 3.7MB.
- 가용 범위(실측): 음양력 ≥1900~2050(2051=0). 절기 = **2000~2027만**(1989~1999=0, 2028+=0,
  정부가 매년 ~2년 앞까지만 갱신). → 절기 KASI 3원 교차는 2000~2027 한정, 그 외 연도는
  lunar↔Skyfield 2원(1955~2049 오차 0.02~0.21분 기검증)으로 자동 폴백.
- 일괄 조회: 음양력 월 일괄(solDay 생략·numOfRows=31), 절기 연 일괄(solMonth 생략). 전체 ~1,840콜.
- 절입시각 교차(Skyfield↔KASI kst) 2000~2027 전수: 672행 중 **3건만** 허용(2분) 초과.
  나머지 669행 ≤2분. 두 계산엔진(Skyfield+lunar-python)은 전 구간 ≤0.03분 일치.

### 2-2. KASI 절기 원본 결함 — 기지 목록 (sajugen/calc/kasi.py: KNOWN_KASI_TERM_DEFECTS)
| 연·절기 | KASI | 정정(Skyfield=lunar 일치) | 차이 | 성격 |
|---------|------|---------------------------|------|------|
| 2011 大寒 | 01-21 19:18 | 01-20 19:18 | 1일 | 일자 +1 오타 |
| 2011 立冬 | 11-08 09:26 | 11-08 03:34 | ~6h | 시각 오류 |
| 2015 夏至 | 06-22 01:58 | 06-22 01:37 | 20분 | 시각 오류 |
- 모두 두 계산엔진이 일치하고 KASI만 어긋남 → KASI 입력오류로 확정(2000-02 우수→입춘 라벨오기와 동류).
- 절기 timing 권위 = Skyfield(DE440), 검증 = lunar-python. KASI 절기는 3차 교차참조.
  라이브 명리 엔진(월주·세운 경계)은 Skyfield(solarterms.py)를 쓰므로 이 결함은 **사주 계산에 영향 없음**.
- KASI 절기 정규화는 dateName이 아니라 KST→UTC 시각의 Skyfield 24절기 최근접 매칭으로 절기명 도출
  (라벨오기 자동 흡수). 음력↔양력 date 변환(1차 기준)은 별개로 결함 없음.

### 2-3. 음양력 캐시 전수 무결성 감사 (scripts/verify_kasi_cache.py, 2026-06-10)

**한·중 음력 라벨 상이**: KASI(한국)≠lunar-python(중국) 1,978일/59개년. **오류 아님** — lunar-python은

- 대표: 2012년 KASI=윤3월(한국)·lunar-python=윤4월(중국). 같은 날 일진 동일(壬子) → 날짜무결성 보존.

## 3. 자미두수 동등성 검증

- iztro-py(현행) ↔ iztro(JS 원본, Node 사이드카=테스트 전용) 무작위 100건+ 명반 전수 대조(12궁 전 성요·사화·대한).
- fortel-ziweidoushu(중주파)로 독립 표본 대조 — 유파 차이로 인한 불일치는 "유파 차이"로 분류, 동일 유파 내 불일치만 결함.
- iztro-py에 fixLeap/yearDivide config 미구현 시: 입력 단계(normalize)에서 음력월 보정 후 전달하는 폴백.
- 불일치 시: 원본 iztro(JS) 값 기준 판정, 케이스 골든셋 등록, iztro-py 패치 또는 사이드카 전환 검토.

## 4. 불일치 처리 정책 (런타임)

1. 3원 중 1원이라도 불일치 → 주문 상태 CALC_MISMATCH, 자동 진행 차단, 관리자 화면 적색 경고 + 어느 소스가 어떻게 다른지 표시.
   - 단, **두 계산엔진(lunar-python↔Skyfield)이 일치하고 KASI 절기만 어긋난 행이 기지 결함 목록
     (KNOWN_KASI_TERM_DEFECTS)에 있으면** 차단하지 않고 '기지 KASI 원본오류'로 분류·기록(kasi_defect_known).
     근거: 절기 timing 권위는 Skyfield(검증 lunar-python)이고 KASI는 3차 참조 — KASI 오타로 유료주문을
     차단함은 부당. 가드 완화가 아니라 불일치 출처(우리 계산 vs KASI 데이터)의 정확한 분류임.
   - 목록에 없는 새 KASI 절기 불일치(미지)는 여전히 실패로 surface(kasi_unknown_mismatches) → 조사 후
     실데이터 확인되면 목록 등록(테스트 test_full_cache_defect_inventory가 목록=실측 일치 회귀 고정).
   - 두 계산엔진 간 불일치는 예외 없이 차단(우리 계산 결함 신호).
2. 절입 ±2분 이내 출생 → 자동 판정 금지, 관리자 확인 플래그(분 단위 반올림 모호성).
3. 윤달 출생 → 15일 분할 자동 적용 + 리포트 고지 + 관리자 화면에 윤달 배지 표시.
4. 모든 불일치는 골든 스냅샷(tests/golden/)에 기록해 회귀 자산화.

## 5. 명리 메인 + 자미 보완 통합 규칙 (해석 계층)

- 최종 권위(총평·시기·길흉): 명리. 자미는 12궁 영역 서술·기질 묘사 공급 엔진으로 한정.
- 자미 대한은 "같은 시기를 다른 각도에서 본 참고 서사"로만 병기, 독립 시기 판정 금지.
- 상충 시: 명리 우선 고정, 자미 상충 서술 제외/강도 인하. 고객 문구는 "모순" 금지, 층위 차이로 재서술
  ("명리로 본 큰 흐름은 X. 자미 관점에서는 같은 시기에 Y의 결이 나타나는데, X가 진행되는 과정의 국면으로 읽습니다").
- 합치 시: "두 체계가 같은 방향" 교차 확인 문구 적극 활용.
- 예측 정확도 주장 전면 금지(양 체계 모두 — 린성쉬안 연구 근거). 자미 권위 문구는 역사적 권위(조선 김치 『심곡비결』)만.
