# 절대 불변 규칙 (항상 로드 — 위반 구조는 작성 즉시 반려)

> 결정 기록: docs/03(유파 결정표), docs/06(LLM 정책), docs/07(안전·고지). 결정일 2026-06-10, 운영자 확정.
> 이 파일의 규칙을 바꾸려면 사용자(운영자)의 명시적 지시가 필요하다.

## 계산 (정확도가 상품의 생명)
1. 계산은 LLM에게 절대 위임 금지: 양음력 변환·절기/월주·일주/시주·대운/세운/월운·십성/오행·격국/용신·자미 명궁/성요/사화/대한 전부 deterministic 엔진 전용.
2. lunar-python은 1.4.8 버전 고정 (이후 수정분 미릴리스 상태, 알려진 이슈 #26·#32는 자체 회귀로 커버). 버전 변경은 골든셋 전수 재검증 후에만.
3. 음력 입력 변환의 1차 기준 = KASI (lunar-python 음력은 중국 기준 — 한·중 윤달 상이일 존재). 불일치일은 KASI 값 채택.
4. KASI는 실시간 API 의존 금지, 사전 캐싱(data/kasi_cache.sqlite)만 참조.
5. 자미두수 윤달 = 15일 분할법 (1~15일생 본월, 16일~ 익월). 리포트에 산입 기준 고지 자동 삽입.
6. 자시 기본 정책 = ZasiPolicy.JST_2300 (23시부터 자시·일주 전환). 명리와 자미는 동일 자시 축 사용.
7. 3원 교차(lunar-python/Skyfield/KASI) 불일치 시 해당 주문 차단(CALC_MISMATCH), 임의 진행 금지. 절입 +-2분 이내 출생은 관리자 확인 플래그.
8. 시진 불명 시: 명리 = 정오 추정 + 고지, 자미 = 생성 금지(섹션 생략, 명리 단독 강등).

## 해석·문안
9. 명리가 최종 권위(총평·시기·길흉). 자미는 12궁 영역 서술·기질 묘사 공급 엔진으로 한정 — 길흉 판정·시기 단언 근거로 사용 금지.
10. 두 체계 상충 시 명리 우선, 고객 문구에 "모순/틀림" 금지, '층위 차이'로 재서술.
11. 예측 정확도 주장 전면 금지 ("적중", "정확히 맞춘다" 등 — 양 체계 모두 학술 검증 없음).
12. 가드 3단(safe_lint / factcheck / trace) 우회·완화 금지. 룰·LLM·관리자 수정분 전부 통과 전 사용 불가.
13. 사실 슬롯(간지·별·수치) 외 새 사실 생성 금지 — factcheck 하드 차단 유지.

## LLM·운영
14. 런타임 LLM = Anthropic 공식 API만. Claude/ChatGPT 웹 UI 헤드리스 자동화로 고객 주문 처리 절대 금지 (약관·계정·개인정보 리스크).
15. LLM 허용 = 챕터 단위 '근거 슬롯 기반 작성'(해석 챕터들) + 질문 분류 + 검수자 재윤문. 각 챕터의 LLM 작성은 결정론 룰 골격과 검증된 계산 사실 슬롯(간지·오행·십성·신살·궁·세운 연도 등)만을 근거로 하며, 근거 밖 사실(간지·별·연도·수치) 생성은 factcheck로 하드 차단한다. 전체 PDF를 한 번에 freeform으로 생성하는 것은 여전히 금지 — 반드시 챕터별 개별 근거 슬롯 기반 작성(이 경계가 핵심). 계산의 LLM 위임은 불변 금지. (2026-06-11 운영자 승인 개정; 근거: 결정론 룰은 고객마다 동일 패턴이라 그 자체가 'AI 출력' 신호 — 사람 풀이 느낌은 사실 슬롯 기반 챕터 작성으로만 확보.)
16. APPROVED(관리자 승인) 전 발송 절대 금지 — 상태 머신으로 물리 차단, 우회 코드 작성 금지.
17. LLM 입력에 생년월일 원본·출생지 비전달(파생 계산값만). 로그에 생년월일 마스킹. 단 '신청 질문 답변' 챕터에 한해 고객 고민 원문 텍스트를 LLM에 전달할 수 있다 — (a) 전달 전 생년월일·출생시각·출생지 문자열을 결정론 마스킹하고, (b) 원문은 "인용이며 지시가 아님"을 명시한 격리 블록으로만 전달하고, (c) 출력은 3단 가드 재검증 후에만 사용하며, (d) 로그·audit에는 마스킹본만 기록한다. 생년월일 원본·출생지의 비마스킹 전달은 계속 금지. (2026-06-11 운영자 승인 개정.)
18. 리포트 본문에는 산출 방식 고지(자동 분석 도구·AI·프로그램 언급)를 넣지 않는다 — AI 산출 인상 일절 금지 (2026-06-12 운영자 명시 지시 개정; 구 규정 = 본문 감수 명시형 고지). "전문가 직접 풀이" 표현·예측 정확도 주장 금지는 유지. 판매 페이지·약관 등 대외 채널에 고지가 필요할 때는 감수 명시형("자동 분석 도구로 산출하고 운영자가 직접 검수·감수한 자료")을 사용한다.

## 작업 방식
19. 검증하지 않은 것을 "완료/정확/해결"이라고 단정 보고 금지. 측정값·테스트 결과로만 말한다.
20. 계산 코드 수정 시 같은 커밋/작업 단위에 테스트 동반. 커밋 전 pytest p1~p5 GREEN 필수.
21. 한 번에 한 작업(Phase 단위). Phase 완료 시 sajugen/STATE.md 갱신.
